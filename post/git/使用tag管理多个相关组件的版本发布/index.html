使用tag管理多个相关组件的版本发布

2018-06-20
git
约 181 字 预计阅读 1 分钟 次阅读

tag标签自增新建脚本

#!/bin/sh
tag=$(git describe --tags `git rev-list --tags --max-count=1`)
version=${tag##*.}
let "version+=1"
newTag=${tag%.*}.${version}
echo 'Create New Tag '$newTag

修改tag版本号的方法

方法一：覆盖

  1. 已有v1.0.2.8要覆盖该版本
git tag -f v1.0.2.8
  1. 服务器已有v1.0.2.8，强制推到服务器
git push origin -f v1.0.2.8
  1. 同步服务器：获取服务器刚刚的v1.0.2.8
git fetch -–tag

方法：删除分支

  1. 删除本地版本
git tag -d v1.0.2.8
  1. 删除服务器上的分支(用空版本覆盖)
git push origin :v1.0.2.8
  1. 服务器获取刚刚的v1.0.2.8
git fetch –-tag
